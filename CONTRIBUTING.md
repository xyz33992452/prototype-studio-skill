# Contributing

Thanks for helping improve `prototype-studio`.

## Ownership

This repository remains owned and maintained by the original author.

External contributors are welcome to:

- read and evaluate the skill
- open issues
- fork the repository
- propose changes through pull requests
- discuss improvements in GitHub conversations

## Good Contribution Areas

- improve prototype generation guidance
- improve requirements-note structure
- add stronger example prototypes
- improve navigation consistency rules
- improve review and QA guidance
- fix unclear or inconsistent instructions

## Before You Open a Pull Request

Please try to keep changes:

- focused
- easy to review
- consistent with the current skill architecture
- backward compatible when possible

## Suggested PR Style

In your pull request, please describe:

1. what problem you are solving
2. what files you changed
3. how the change improves prototype generation or review workflows
4. whether the change affects examples, schema, or behavior expectations

## Notes

- Large rewrites are better discussed in an issue first.
- If you are unsure, open an issue before implementing a major direction change.
