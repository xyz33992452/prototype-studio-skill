# Page Modes

This skill does not use a rigid template library.

Instead, it uses page modes. A page mode is a structural pattern with clear intent, but flexible layout.

## Core Rule

Choose page mode by the page's communication job, not by habit.

A page can borrow small ideas from another mode, but should still have one primary mode.

## `list-page`

Best for:

- browsing many records
- search and filter tasks
- queue handling
- operational management

Primary job:

- help users scan, narrow, compare, and act on multiple items

Typical regions:

- title and context
- search and filters
- bulk or primary actions
- results area
- pagination or tabs

Low-fi emphasis:

- visible hierarchy of filters and results
- mock rows or cards
- action placement clarity

High-fi emphasis:

- stronger density control
- refined table/card states
- sort, filter, hover, and selection feedback

## `detail-page`

Best for:

- single-record understanding
- profile or entity review
- case, order, or ticket detail

Primary job:

- explain one entity clearly and support key follow-up actions

Typical regions:

- title and status
- summary block
- grouped details
- related information
- action area

Low-fi emphasis:

- section grouping
- relationship between summary and details
- obvious action locations

High-fi emphasis:

- visual hierarchy
- usable secondary data presentation
- timeline, tabs, or side detail patterns where appropriate

## `form-page`

Best for:

- create/edit operations
- settings and configuration
- submissions and approvals

Primary job:

- let users complete structured input confidently

Typical regions:

- title and instructions
- grouped form blocks
- helper and validation text
- action footer or sticky actions

Low-fi emphasis:

- field grouping
- label clarity
- required vs optional inputs

High-fi emphasis:

- polished control states
- validation behavior
- stronger spacing and form rhythm

## `dashboard-page`

Best for:

- executive views
- operational overviews
- status monitoring

Primary job:

- communicate state quickly and support drill-down

Typical regions:

- key metrics
- charts
- trends or alerts
- activity or data modules

Low-fi emphasis:

- metric hierarchy
- module priority
- placeholder charts with labels

High-fi emphasis:

- polished metrics and charts
- readable density
- meaningful visual contrast

## `workspace-page`

Best for:

- dense systems where users perform ongoing tasks
- console-like tools
- editor and operator interfaces

Primary job:

- support sustained task execution with clarity

Typical regions:

- navigation
- main work area
- supporting panel
- contextual actions

Low-fi emphasis:

- work area boundaries
- panel responsibilities
- task sequence clarity

High-fi emphasis:

- efficient dense layout
- better state handling
- refined panel and toolbar interactions

## `wizard-page`

Best for:

- sequential tasks
- onboarding
- setup or review flows

Primary job:

- move users through a defined sequence with confidence

Typical regions:

- stepper
- focused content area
- guidance
- next/back/save actions

Low-fi emphasis:

- sequence logic
- step clarity
- next-step expectations

High-fi emphasis:

- reassuring progress design
- clearer validation and transition cues

## `flow-page`

Best for:

- process explanation
- business flow
- user journey
- approval logic

Primary job:

- make sequence, branching, and ownership understandable

Typical regions:

- nodes or stages
- connectors
- branch logic
- ownership or role notes

Low-fi emphasis:

- readable sequence
- branch logic
- role labels

High-fi emphasis:

- better visual grouping
- cleaner emphasis and directionality

## `intro-page`

Best for:

- product introductions
- value proposition pages
- capability overviews

Primary job:

- orient the audience and frame the story

Typical regions:

- headline statement
- value summary
- key sections or features
- next-step or CTA

Low-fi emphasis:

- message hierarchy
- section framing

High-fi emphasis:

- stronger storytelling
- polished supporting visuals

## `comparison-page`

Best for:

- alternatives
- old vs new
- package/plan comparison
- solution comparison

Primary job:

- help users understand tradeoffs and choose

Typical regions:

- comparison dimensions
- side-by-side structure
- recommendation summary

Low-fi emphasis:

- dimension clarity
- decision clarity

High-fi emphasis:

- easier scanning
- stronger recommendation focus

## `empty-state-page`

Best for:

- no results
- first-use state
- pre-setup state
- blocked or disabled state

Primary job:

- explain the state and move the user forward

Typical regions:

- explanation
- next action
- optional education

Low-fi emphasis:

- clarity of reason and next step

High-fi emphasis:

- stronger tone and guidance without overdesign
