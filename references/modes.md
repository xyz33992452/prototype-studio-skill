# Prototype Studio Modes

## Quick Selector

Use this table when choosing the mode.

| Situation | Best mode | Why |
|---|---|---|
| Early idea, quick alignment, rough scope | Low-Fi Wireframe | Structure matters more than polish |
| Product review, PRD discussion, handoff | Annotated Review Prototype | Needs explanation, not just visuals |
| Demo, showcase, stakeholder persuasion | Hi-Fi Clickable Prototype | Realism and confidence matter |
| Workflow, logic, navigation, ownership | Flow / IA Artifact | Relationships matter more than single screens |
| Mixed audience or multi-step review | Multi-Surface Prototype Pack | One artifact will not cover all needs |

## Mode Details

### Low-Fi Wireframe

Include:
- screen title
- major layout regions
- primary actions
- navigation and hierarchy
- placeholders for data-heavy modules

Avoid:
- heavy branding
- detailed visual styling
- decorative imagery

### Annotated Review Prototype

Include:
- page goal
- target user or reviewer
- key modules
- important states
- interaction notes
- assumptions or unresolved questions

Good annotation categories:
- purpose
- interaction
- state
- data
- permission
- dependency

### Hi-Fi Clickable Prototype

Include:
- polished screen composition
- believable content
- key primary journeys
- useful hover, click, or navigation affordances

Avoid:
- fake depth without interaction meaning
- heavy motion that distracts from the product story

### Flow / IA Artifact

Include:
- entry points
- main nodes
- transitions
- decisions or branches
- ownership where relevant

Suitable artifacts:
- user flow
- page map
- business process map
- feature relationship map

### Multi-Surface Prototype Pack

Recommended bundle:
- one low-fi or IA artifact for structure
- one annotated screen for review
- one higher-fidelity screen for demo value

Use this mode when the audience includes both builders and decision-makers.

## Notes for Product Managers

For PM-focused requests, prefer one of these:

- `Annotated Review Prototype` for requirement alignment
- `Flow / IA Artifact` for logic-heavy subjects
- `Multi-Surface Prototype Pack` for important reviews

Do not default to `Hi-Fi Clickable Prototype` unless:
- persuasion is part of the goal
- the user explicitly wants a polished demo
- structure is already settled
